# 🌱 GitHub Pages から Netlify に移行する理由と背景

## ■ GitHub Pages 側で起きた問題
- GitHub Actions のランナーが **ubuntu‑24.04** に更新  
- これにより **Ruby がプリインストールされなくなった**  
- Jekyll をビルドするには Ruby を自前でインストールする必要がある  
- その結果、あなたの Actions が壊れた

つまり、**GitHub Pages の構造的な変更が原因で、Jekyll サイトのビルドが不安定になった**。

---

# 🌿 Netlify に移行すると何が変わるのか

## ■ Netlify の強み
Netlify は Jekyll をビルドするための環境を **最初から持っている**。

- Ruby 自動セットアップ  
- Bundler 自動  
- `bundle install` 自動  
- `jekyll build` 自動  
- `_site` をそのまま公開  
- HTTPS 自動  
- DNS も Netlify 管理（あなたの southwiseman.blog は Netlify で購入したドメイン）

つまり、**GitHub Pages で起きていた Ruby・DNS の問題がすべて解消される**。

---

# 🔧 昨晩まとめた「移行の実際の作業」

## ① Netlify に GitHub リポジトリを接続
Netlify → Sites → Add new site → Import from GitHub  
→ あなたの Jekyll リポジトリを選択

## ② Build 設定（最重要）
Build command:

```
bundle exec jekyll build
```

Publish directory:

```
_site
```

Ruby バージョン指定は不要。

## ③ `_config.yml` の調整
GitHub Pages 用の設定が残っていると Netlify が誤って **github-pages gem** を使ってしまう。

そのため：

- `whitelist:` は削除  
- `plugins:` は最小限（`jekyll-include-cache` のみ）  
- `remote_theme:` はそのままで OK（Minimal Mistakes は対応済み）

## ④ ドメイン設定
あなたの southwiseman.blog は Netlify ドメインなので：

1. Netlify → Domains  
2. southwiseman.blog を選択  
3. **Set as primary domain**  
4. HTTPS 自動有効化

GitHub Pages の DNS エラー（NotServedByPagesError）はもう関係なし。
